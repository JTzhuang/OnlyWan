Change 3 — Fix avi_writer.h include guard.
In examples/cli/avi_writer.h:
  Line 8: `#ifndef __AVI_WRITER_H__`  →  `#ifndef AVI_WRITER_H`
  Line 9: `#define __.AVI_WRITER_H__`  →  `#define AVI_WRITER_H`
  Line 79: `#endif /* __AVI_WRITER_H__ */`  →  `#endif /* AVI_WRITER_H */`
The leading `__` identifiers are reserved by the C++ standard (ISO C++ 17.6.4.3). The `.` in the define macro caused the guard to never activate on re-inclusion.
