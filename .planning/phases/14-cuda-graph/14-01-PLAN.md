<task type="auto">
  <name>Task 1: CG-01 + OP-01 — 缓冲区持久化优化 + Flash Attention 自动启用</name>
  <files>src/ggml_extend.hpp, src/api/wan-api.cpp</files>
  <action>
  **CG-01 — 优化 GGMLRunner::compute 中的重复 graph 重建：**

  当前问题：即使 `free_compute_buffer_immediately=false` 保留了 allocator，每次 compute() 仍调用 `reset_compute_ctx()` + `get_compute_graph()` + `ggml_gallocr_alloc_graph()`，导致每步都重建 GGML context 和 graph。

  在 `GGMLRunner` 类中（src/ggml_extend.hpp）：
  1. 添加成员变量 `bool graph_structure_stable = false;` 和 `struct ggml_cgraph* cached_graph = nullptr;`
  2. 添加方法 `void set_graph_structure_stable(bool stable)` 设置标志
  3. 修改 `compute()` 方法：当 `graph_structure_stable && compute_allocr != nullptr && cached_graph != nullptr` 时，跳过 `alloc_compute_buffer` / `reset_compute_ctx` / `get_compute_graph` / `ggml_gallocr_alloc_graph`，直接执行 `copy_data_to_backend_tensor()` + `ggml_backend_graph_compute()`
  4. 在首次成功 compute 后（且 `!free_compute_buffer_immediately`），缓存 `cached_graph = gf`
  5. 在 `free_compute_buffer()` 中清除 `cached_graph = nullptr`

  **OP-01 — Flash Attention 自动启用：**

  在 `src/api/wan-api.cpp` 的模型加载函数中（`wan_load_model_impl` 或等效位置），在创建各 runner 之后、返回之前：
  1. 检测后端类型：使用 `ggml_backend_is_cpu()` 判断 — 如果不是 CPU 后端，则自动启用 flash attention
  2. 对 `wan_runner`、`vae_runner` 调用 `set_flash_attention_enabled(true)`
  3. 对 `clip_runner`（如果非 null）也调用 `set_flash_attention_enabled(true)`
  4. 添加日志：`LOG_INFO("Auto-enabled flash attention for non-CPU backend")`

  注意：T5Embedder 不继承 GGMLRunner 的 flash_attn 接口，跳过。
  </action>

  </verify>
  <done>
