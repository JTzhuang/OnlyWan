<objective>
Expose the internal vocab directory mechanism as a public C API function `wan_set_vocab_dir`, add load-time validation in `wan_load_model`, and wire `--vocab-dir` into the CLI. After this plan, `WAN_EMBED_VOCAB=OFF` builds can load vocab files from an external directory via the public API and CLI.

Purpose: PERF-01 requires vocab to be loaded at runtime from external files; this plan completes the public surface that makes that possible. ENCODER-01/02 (T5/CLIP tokenizers) depend on `g_vocab_dir` being set before model load. API-03/04 (T2V/I2V generation) fail with empty tokenizer output if vocab dir is absent.

Output: Updated vocab.h, vocab.cpp, wan.h, wan-api.cpp, main.cpp, examples/cli/CMakeLists.txt.
</objective>
