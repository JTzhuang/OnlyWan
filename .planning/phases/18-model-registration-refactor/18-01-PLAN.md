<objective>
Migrate model version registration from test-only enum-based factories to production-ready infrastructure using compile-time macros and string-based versioning. This enables future public API exposure of model factories and simplifies version management across the codebase.

Purpose: Establish a scalable, maintainable registration system that separates version identity (strings) from implementation details (enums), enabling future features like dynamic model loading and plugin systems.

Output: Production-ready registry infrastructure in src/, all 14 model variants registered globally, test suite updated to use string-based lookup, zero breaking changes to existing APIs.
</objective>

## Architecture Overview

```
src/model_registry.hpp
├─ ModelRegistry (singleton)
│  ├─ register_factory<ModelType>(version_string, factory_fn)
│  ├─ create<ModelType>(version_string, backend, offload, tensor_map, prefix)
│  └─ has_version(version_string)
├─ FactoryFn type alias
└─ REGISTER_MODEL_FACTORY macro (compile-time registration)

src/model_registry.cpp
├─ Global registry instance
├─ std::map<string, FactoryFn> storage
└─ std::mutex for thread safety

src/model_factory.hpp
├─ CLIPFactory (ModelFactory<CLIPTextModelRunner, CLIPVersion>)
├─ T5Factory (ModelFactory<T5Runner, T5Version>)
└─ VAEFactory (ModelFactory<VAE, SDVersion>)

src/model_factory.cpp
└─ REGISTER_MODEL_FACTORY macro invocations for all 9 variants
```

- No changes to public API (wan.h) — registry is internal infrastructure
- All existing tests continue to work with minimal modifications
