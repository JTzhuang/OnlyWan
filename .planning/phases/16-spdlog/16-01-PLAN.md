# Phase 16: 集成 spdlog 日志系统 - 实施计划

## 目标
将 `wan-cpp` 内部的日志系统全面替换为 `spdlog`。通过自定义 Sink 实现日志分发（终端彩色输出 + 用户 C 回调），支持动态级别调整，并重构 C API 接口。

## 成功标准
1. `wan-cli` 默认输出 `INFO` 级别及以上的彩色日志。
2. 支持 `WAN_LOG_LEVEL` 环境变量控制初始级别。
3. 提供 `wan_set_log_level` API 动态修改级别。
4. 重构 `wan_set_log_callback`，回调能接收到 `spdlog` 格式化后的完整字符串。
5. 所有日志操作线程安全（使用 `_mt` 组件）。

## 详细步骤

### 步骤 1: 重构公共头文件 `include/wan-cpp/wan.h`
1. 修改 `wan_log_cb_t` 为：
   ```c
   typedef void (*wan_log_cb_t)(const char* text, void* data);
   ```
2. 添加级别设置函数：
   ```c
   void wan_set_log_level(int level);
   ```

### 步骤 2: 在 `src/util.cpp` 实现自定义 Sink
1. 定义 `sd_callback_sink` 类，继承自 `spdlog::sinks::base_sink<std::mutex>`。
2. 在 `sink_it_` 方法中，调用全局存储的回调函数。

### 步骤 3: 优化日志宏定义
为了让 `spdlog` 自动获取文件名和行号，我们需要改变宏的使用方式。
从：
```cpp
#define LOG_INFO(format, ...)  log_printf(SD_LOG_INFO, __FILE__, __LINE__, format, ##__VA_ARGS__)
```
改为类似：
```cpp
#define LOG_INFO(...) SPDLOG_LOGGER_INFO(get_wan_logger(), __VA_ARGS__)
```
或者保留 `log_printf` 作为一个 shim，但内部使用 `spdlog::source_loc`。

### 步骤 4: 编译系统检查
确保 `thirdparty/spdlog` 被正确包含。

## 风险
- C API 的不兼容变更可能会破坏现有的外部绑定（如果有的话），但根据指令，这是允许的。
