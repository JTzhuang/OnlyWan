# Phase 16: 集成 spdlog 日志系统

## 目标
将 `wan-cpp` 内部的日志系统替换为 `spdlog`。支持通过环境变量控制日志级别，并为 CLI 工具提供默认的彩色终端输出，同时保持 C API 的回调兼容性。

## 需求
- **LOG-01**: 集成 `spdlog` (header-only) 到项目构建系统。
- **LOG-02**: 在 `src/util.cpp` 中重构 `log_printf` 以使用 `spdlog` 后端。
- **LOG-03**: 支持 `WAN_LOG_LEVEL` 环境变量设置日志级别（debug, info, warn, error）。
- **LOG-04**: 保持 `sd_set_log_callback` 的 C API 兼容性，将 `spdlog` 消息转发至回调。

## 依赖
- Phase 15 完成

## 成功标准
1. `wan-cli` 运行时默认在终端输出 INFO 级别的日志。
2. 设置 `WAN_LOG_LEVEL=debug` 时，能够看到详细的调试日志。
3. 如果外部应用程序设置了日志回调，回调仍能收到日志消息。
4. 编译过程能够正确拉取或引用 `spdlog`。

## 计划内容

### 16-01-PLAN.md — 基础 spdlog 集成与 util 日志宏重写

#### 步骤 1: 修改 CMakeLists.txt 以引入 spdlog
- 在 `thirdparty/CMakeLists.txt` 中使用 `FetchContent` 引入 `spdlog`。
- 将 `spdlog::spdlog` 链接到 `wan-cpp` 目标。

#### 步骤 2: 更新 src/util.h
- 包含 `spdlog/spdlog.h` 和 `spdlog/fmt/ostr.h`。
- 保持 `LOG_DEBUG`, `LOG_INFO` 等宏定义不变，但后端逻辑将发生变化。

#### 步骤 3: 重构 src/util.cpp
- 实现 `get_wan_logger()` 单例，初始化 `spdlog::logger`。
- `log_printf` 实现逻辑：
    - 将 `sd_log_level_t` 映射到 `spdlog::level`。
    - 使用 `spdlog` 的格式化能力（如果可能，或者继续使用 `vsnprintf` 但将结果传递给 `spdlog`）。
    - 检查环境变量 `WAN_LOG_LEVEL` 进行初始化。
    - 如果存在 `sd_log_cb`，则在 `spdlog` 记录后（或通过自定义 sink）调用它。

#### 步骤 4: 验证
- 编译项目。
- 运行 `wan-cli` 确认日志输出。
- 测试环境变量 `WAN_LOG_LEVEL` 是否生效。
