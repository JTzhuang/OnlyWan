# Quick Task 260328-vj7: Add --model-path Parameter to benchmark_inference

## Objective
Add `--model-path` parameter support to the benchmark_inference tool to enable loading real model weights from file for complete inference benchmarking (not just compute graph construction).

## Context
The benchmark_inference tool currently creates runners with empty weights. Adding model_path support allows users to benchmark with actual model weights loaded from disk.

## Tasks

### Task 1: Add model_path parameter to all benchmark functions
- Add `const std::string& model_path = ""` parameter to benchmark_clip, benchmark_t5, benchmark_vae, benchmark_transformer
- Modify weight loading logic: if model_path is not empty, call `runner->load_from_file(model_path)`, otherwise use `alloc_params_buffer()`
- Add error handling for file loading failures

### Task 2: Add command-line argument parsing
- Add `std::string model_path = "";` variable in main()
- Parse `--model-path PATH` argument
- Update help text to document the new option

### Task 3: Pass model_path to benchmark functions
- Update all benchmark function calls in main() to pass model_path parameter
- Maintain backward compatibility (empty string when not specified)

## Success Criteria
- All 4 benchmark functions accept model_path parameter
- Command-line parsing works correctly
- Help text includes --model-path documentation
- Backward compatible (works without --model-path)
- Code compiles without errors
