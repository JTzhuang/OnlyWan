<interfaces>
<!-- Key patterns the executor needs to understand -->

GGMLRunner::compute() pattern (ggml_extend.hpp:2058-2105):
- Each compute() call: alloc_compute_buffer -> build graph -> gallocr_alloc -> copy data -> backend_graph_compute -> free buffer
- free_compute_buffer_immediately=true by default — buffer freed after every call
- This means CUDA graph cannot be reused across denoising steps

Denoising loop (wan-api.cpp:547-595):
- Euler loop runs `steps` iterations (default 20)
- Each step: 2x wan_runner->compute() (cond + uncond for CFG)
- Each compute() rebuilds graph from scratch, reallocates buffer
- Total: 40 graph builds + 40 buffer allocs for 20-step generation

GGML CUDA Graph support (ggml-cuda.cu:3918+):
- GGML already has CUDA graph capture/replay via GGML_CUDA_USE_GRAPHS
- Activated when graph structure is stable across calls
- wan-cpp defeats this by freeing/reallocating between steps

Attention pattern (wan.hpp, common_block.hpp):
- Uses ggml_ext_attention_ext() which dispatches to ggml_flash_attn_ext when enabled
- flash_attn_ag exists but must be explicitly set
- Q/K/V projections are separate Linear layers (no fused QKV)

WanAttentionBlock forward (wan.hpp:1533-1581):
- LayerNorm -> modulate -> SelfAttn -> modulate -> CrossAttn -> LayerNorm -> modulate -> FFN -> modulate
- 6 modulation parameters per block, applied via separate add/mul ops
- FFN: Linear -> GELU -> Linear (no fused MLP)
