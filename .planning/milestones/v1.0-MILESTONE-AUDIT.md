## Tech Debt Summary

### Phase 3 — Public API
- `WanImage::*` and `WanVideo::*` methods declared in wan-internal.hpp but never defined — dead declarations (linker error if called)
- `wan_generate_video_t2v/i2v` (legacy flat signatures) permanently return `WAN_ERROR_UNSUPPORTED_OPERATION` — public API surface includes broken entry points with no deprecation notice

### Phase 4 — Examples
- `avi_writer.h` comment says "stub / placeholder data" — stale, implementation is complete
- `-v` flag collision: acts as `--version` when first arg, preventing `--verbose` via `-v`

### Phase 5 — Encoders
- Vocabulary headers embedded (~85MB total) — consider memory-mapped I/O for production

### Phase 7 — Wire Core Model
- `WanBackend ctx` in `wan_load_model` allocates 256MB buffer but is never used in generation — freed on `wan_free()` but wastes memory during inference
- ROADMAP Phase 7 success criteria reference `wan_loader.cpp`/`preprocessing.hpp` — actual implementation uses `wan-api.cpp`/`ggml_extend.hpp` (intentional ODR fix); ROADMAP text is inaccurate

### Phase 8 — Generation
- `progress_cb` wired in CLI but never invoked inside generation functions — users see no progress during multi-minute inference
- `negative_prompt` silently ignored — CFG unconditional pass hardcodes `""` instead of `params->negative_prompt`

### Total: 10 tech debt items across 5 phases
